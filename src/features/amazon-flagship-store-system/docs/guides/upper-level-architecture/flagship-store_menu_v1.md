# 亚马逊旗舰店子系统 · 菜单架构 v1.0

> **文档日期**：2026-04-22
> **本图回答**：本子系统给"哪个角色"看"哪些功能"？

---

## 一、设计原则

1. **能力 → 菜单一一对照**：每条菜单必须能在 L1 能力图（`flagship-store_capability_architecture.md`）找到对应能力；找不到的能力要补，没人用的菜单要砍。
2. **业务方语言优先**：菜单命名直接复用业务方原话（"场景 / 页面 / 重点品 / 旗舰店占比"），不引入 IT 黑话。
3. **看板与工具分开**："看数据"在【旗舰店看板】里聚合（7 大 Tab）；"做事"分散到【Tag 中心 / 联盟中心 / 直播中心 / 站点与场景 / 重点品与目标】。
4. **强约束写入菜单**：Tag 命名规范、联盟佣金比例、页面-场景映射快照——全部要有显式菜单入口，避免回到 Excel 时代。
5. **MVP 优先级标注**：每个三级菜单后标 `P0` / `P1`，与 README §4 范围保持一致。
