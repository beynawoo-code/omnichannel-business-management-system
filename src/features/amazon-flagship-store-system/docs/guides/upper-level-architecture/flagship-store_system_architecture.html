💡 v1.0 系统架构的三大核心洞察
1. 数据中台是系统的"心脏"

所有指标只在 L2 中台计算一次，前端只展示。比率指标必须 sum/sum 重算（K3）；映射版本快照解决场景迁移历史归属（K2）。这是替代"Excel + DeepSeek 手工流"的根本能力。

2. 跨子系统数据 = domain 中转

本系统不直接 import amazon-sales / amazon-ads / OC 的代码，全部通过 src/domain/ 交换。ASIN-SKU 主数据、销售明细、广告投放结果都在 domain 共享。

3. AI 嵌入业务场景，不做独立入口

4 类 AI 能力（命名纠错 / 异常识别 / 复盘撰写 / 自然语言搜 SKU）全部嵌入 8 大模块的对应菜单内，遵循 OC 系统的 AI 嵌入哲学，避免出现"智能助手"孤立菜单。
